📖 Breve historia de los números aleatorios en Python

Desde sus primeras versiones, Python incorporó la generación de números seudoaleatorios a través del módulo random, basado en el algoritmo Mersenne Twister. Este generador, diseñado en 1997 por Makoto Matsumoto y Takuji Nishimura, ofrece un período enorme de 219937-1 y una distribución uniforme, siendo ideal para simulaciones, juegos y pruebas. La comunidad adoptó rápidamente random por su simplicidad: random.random(), randint o choice se volvieron herramientas cotidianas. Sin embargo, esta aleatoriedad es predecible si se conoce la semilla, lo que la hace inadecuada para contextos sensibles.

Con la madurez de Python y el auge de la ciberseguridad, en la versión 3.6 se introdujo el módulo secrets, diseñado específicamente para generar tokens criptográficamente seguros. Mientras que random usa una semilla determinista, secrets recurre a fuentes de entropía del sistema operativo (/dev/urandom en Linux, CryptGenRandom en Windows). Esto marcó un hito: por primera vez Python ofrecía en la librería estándar una forma clara y recomendada de crear contraseñas, tokens de autenticación o claves seguras. La distinción entre "aleatoriedad para simulación" y "aleatoriedad para seguridad" quedó formalizada.

El ecosistema científico también impulsó la evolución. NumPy, la biblioteca fundamental para cómputo numérico, incorporó su propio submódulo numpy.random con generadores vectorizados y distribuciones estadísticas (normal, exponencial, etc.). En 2019, con NumPy 1.17, se adoptó un sistema basado en generadores (Generator y default_rng) que permiten mayor control, velocidad y la posibilidad de usar distintos algoritmos (como PCG64). Hoy, Python ofrece un ecosistema completo: desde random para scripting, secrets para seguridad, hasta np.random.Generator para big data y machine learning, consolidando más de dos décadas de evolución en aleatoriedad.

💡 Nota sobre diferentes tamaños: Para operar matrices de diferente tamaño, se puede usar broadcasting implícito (por ejemplo, sumar escalar o vector fila a matriz) o alinear dimensiones con reshape. Para producto matricial, la condición clásica: (n x m) @ (m x p) = (n x p). Para elemento a elemento las dimensiones deben ser idénticas o transmisibles (broadcast).
